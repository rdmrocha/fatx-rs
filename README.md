# fatx-rs

[![Release](https://img.shields.io/github/v/release/joshuareisbord/fatx-rs?include_prereleases&sort=semver)](https://github.com/joshuareisbord/fatx-rs/releases)
[![CI](https://img.shields.io/github/actions/workflow/status/joshuareisbord/fatx-rs/ci.yml?branch=main&label=CI)](https://github.com/joshuareisbord/fatx-rs/actions/workflows/ci.yml)
[![License](https://img.shields.io/badge/License-Apache_2.0-blue.svg)](LICENSE)

A Rust toolkit for reading, writing, and mounting FATX/XTAF file systems used by Xbox and Xbox 360 consoles. Connect a drive via USB to macOS and browse, extract, modify, or mount files directly in Finder.

## Supported Formats

| Format | Console | Endianness | Auto-detected |
|--------|---------|------------|---------------|
| **FATX** | Original Xbox | Little-endian | Yes |
| **XTAF** | Xbox 360 | Big-endian | Yes |

Both FAT16 and FAT32 entry sizes are handled automatically based on partition size.

## Features

**File Operations**
- Full read/write support: list, read, write, create directories, delete, rename, recursive delete
- Recursive directory copy with automatic macOS metadata filtering
- Automatic partition detection at standard Xbox and Xbox 360 offsets
- Interactive guided mode when run with no arguments
- TUI file browser for navigating, uploading, downloading, and cleaning up volumes
- JSON output mode (`--json`) for scripting and automation
- macOS metadata cleanup command (scan + delete `.DS_Store`, `._*`, etc.)

**Finder Mount**
- Mount FATX volumes in Finder via a local NFS server
- Guided mount: `sudo fatx mount` with no arguments detects drives and mounts automatically
- Read and write files through Finder drag-and-drop
- Clean unmount on Ctrl+C with signal handling

**Performance**
- macOS-native I/O: `F_NOCACHE`, `F_RDAHEAD`, device-optimal alignment via `nix` ioctls
- Free-cluster bitmap for fast allocation (bit scanning vs FAT entry iteration)
- Next-fit allocation with `prev_free` hint (matching Linux FAT driver pattern)
- Dirty-range FAT tracking (flush only modified pages, not the entire FAT)
- Cached free cluster count (O(1) stats)
- Concurrent NFS reads via `parking_lot::RwLock` + `quick_cache` bounded caches
- Zero-copy NFS responses via `bytes::Bytes`

**Testing**
- Automated test suite with integration, CLI, optimization, and property-based tests
- File-backed test images generated by `fatx-mkimage` (no hardware required)
- Property-based roundtrip tests via `proptest`
- CLI integration tests for every non-interactive command

## Install

### One-liner (macOS)

Downloads the latest release binary:

```bash
curl -fsSL https://raw.githubusercontent.com/joshuareisbord/fatx-rs/main/install.sh | bash
```

### Build from Source

Requires Rust (stable). The setup script installs Rust if needed, runs the test suite, and builds:

```bash
git clone https://github.com/joshuareisbord/fatx-rs.git
cd fatx-rs
bash setup.sh
```

Or build manually:

```bash
cargo build --release
```

Produces a single binary: `target/release/fatx`. All functionality is accessed through subcommands.

## Quick Start

```bash
# Find your Xbox drive
diskutil list | grep external

# Unmount macOS's hold on it
diskutil unmountDisk /dev/diskN

# Scan for Xbox partitions
sudo fatx scan /dev/rdiskN

# List files
sudo fatx ls /dev/rdiskN --partition "360 Data" /

# Interactive mode (guided — prompts for everything)
sudo fatx
```

## Usage

### Interactive Mode

Run with no arguments for guided drive detection and partition selection:

```bash
sudo fatx
```

### Scan for Partitions

```bash
sudo fatx scan /dev/rdisk4
```

Probes standard Xbox/Xbox 360 partition offsets and reports which ones have valid FATX/XTAF headers.

### List Files

```bash
sudo fatx ls /dev/rdisk4 --partition "360 Data" /
sudo fatx ls /dev/rdisk4 --partition "360 Data" /Content -l
```

### Read a File

```bash
# Print to stdout
sudo fatx read /dev/rdisk4 --partition "360 Data" /name.txt

# Save to local file
sudo fatx read /dev/rdisk4 --partition "360 Data" /name.txt -o name.txt
```

### Write a File

```bash
sudo fatx write /dev/rdisk4 --partition "360 Data" /hello.txt -i hello.txt
```

### Create a Directory

```bash
sudo fatx mkdir /dev/rdisk4 --partition "360 Data" /MyFolder
```

### Delete

```bash
# Single file or empty directory
sudo fatx rm /dev/rdisk4 --partition "360 Data" /hello.txt

# Recursive delete (directories with contents)
sudo fatx rmr /dev/rdisk4 --partition "360 Data" /Content
```

### Copy a Directory

```bash
# Recursively copy a local directory onto the FATX volume
sudo fatx copy /dev/rdisk4 --partition "360 Data" --from /path/to/local/folder --to /DestFolder
```

macOS metadata files (`.DS_Store`, `._*`, `.Spotlight-V100`, `.Trashes`, `.fseventsd`) are automatically skipped during copy.

### Rename

```bash
sudo fatx rename /dev/rdisk4 --partition "360 Data" /old.txt new.txt
```

### Volume Info

```bash
sudo fatx info /dev/rdisk4 --partition "360 Data"
sudo fatx info /dev/rdisk4 --partition "360 Data" --json
```

### TUI File Browser

```bash
# Guided mode — detects disks and partitions automatically
sudo fatx browse

# Browse a specific partition
sudo fatx browse /dev/rdisk4 --partition "360 Data"
```

Interactive terminal UI for navigating, downloading, uploading, and cleaning up files. Press `c` to scan and remove macOS metadata from the current directory.

### Mount in Finder

```bash
# Guided mode — detects disks, scans partitions, mounts automatically
sudo fatx mount

# Mount a specific partition
sudo fatx mount /dev/rdisk4 --partition "360 Data" -v --mount

# Start NFS server only (no Finder mount)
sudo fatx mount /dev/rdisk4 --partition "360 Data" -v

# Mount a test image
sudo fatx mount test.img -v --mount

# Emergency cleanup if a previous mount went stale
sudo fatx mount --cleanup
```

The mount uses a local NFSv3 server with `soft,intr` options. Ctrl+C cleanly unmounts before exiting. The initial FAT load for large drives (1TB+) takes a few seconds over USB — this is normal.

### Clean Up macOS Metadata

Remove `.DS_Store`, `._*` resource forks, `.Spotlight-V100`, `.Trashes`, and `.fseventsd` from the volume:

```bash
# Preview what would be deleted
sudo fatx cleanup /dev/rdisk4 --partition "360 Data" --dry-run

# Delete them
sudo fatx cleanup /dev/rdisk4 --partition "360 Data"
```

In interactive mode (option 10), the cleanup scans first and asks for confirmation before deleting.

### Create Test Images

Generate FATX/XTAF disk images for testing without hardware:

```bash
fatx mkimage test.img --size 1G --populate        # FATX with sample content
fatx mkimage xbox360.img --size 512M --format xtaf # Xbox 360 format
fatx mkimage test.img --size 1G --force            # overwrite existing
```

### JSON Output

Add `--json` to any command for machine-readable output:

```bash
sudo fatx info /dev/rdisk4 --partition "360 Data" --json
sudo fatx ls /dev/rdisk4 --partition "360 Data" / --json
```

### Hex Dump

```bash
sudo fatx hexdump /dev/rdisk4 --offset 0x80080000 --count 512
```

## macOS Notes

- Use `/dev/rdiskN` (raw device), **not** `/dev/diskN` — raw is significantly faster
- `sudo` is required for raw device access and mounting
- Find your drive: `diskutil list` — look for the unformatted external disk
- Unmount macOS first: `diskutil unmountDisk /dev/diskN`

## Project Structure

Cargo workspace with two crates and a single binary:

| Crate | Binary | Purpose |
|-------|--------|---------|
| `fatxlib` | — | Core library: FATX/XTAF volume I/O, FAT operations, partition detection, platform helpers |
| `fatx-cli` (root) | `fatx` | Single binary: CLI interface, NFS mount server, image generator, TUI browser, JSON output |

## Testing

```bash
cargo test --workspace
```

Tests use file-backed FATX/XTAF images generated by `fatx mkimage`. No hardware required. The suite includes integration tests, CLI tests for every command, optimization regression tests, and property-based roundtrip tests via `proptest`.
